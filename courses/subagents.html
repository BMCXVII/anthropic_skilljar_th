Introduction to Subagents

Subagent คือ ผู้ช่วยเฉพาะทาง ที่คุณ delegate งานให้ มันรันใน context window แยก แล้วส่งเฉพาะสรุปกลับมายัง main thread

⏱️ 4 บท · เริ่มต้น–กลาง 🎯 ใช้กับ Claude Code 📌 สถานะ: ลงทะเบียนแล้ว

คอร์สสั้นกระชับ เรียบเรียงตามลำดับบทเรียนจริง: subagent คืออะไร → สร้างอย่างไร → ออกแบบให้ดี → ใช้ตอนไหนถึงคุ้ม

🧩 Subagent คืออะไร

มองว่าเป็นผู้ช่วยที่คุณส่งงานให้แล้วมันไปทำใน context ของตัวเอง ขั้นตอนกลาง ๆ ระหว่างทาง จะไม่มารกใน main thread — สุดท้ายมันส่งกลับมาแค่ สรุปผล แต่ละ subagent รับ custom system prompt + task description ของตัวเอง จึงทำงานเฉพาะทางได้ดี

🛠️ สร้าง subagent

มีทั้งแบบ built-in และสร้างเอง โดยนิยามเป็นไฟล์ markdown + YAML frontmatter วิธีสร้างผ่านเมนู: /agents → Create new agent → เลือก scope (Project/User) → ให้ Claude ช่วย generate config ให้

🎯 ออกแบบให้ดี — 4 ข้อ

📝

1. Description ที่ดี

ช่วยทั้งให้ Claude เลือก subagent ถูกตัว และช่วยเขียน input prompt ที่ตรง

📤

2. Output format

กำหนดรูปแบบผลลัพธ์ที่อยากได้กลับมาให้ชัด

🚧

3. รายงานอุปสรรค

ให้ subagent บอกเมื่อเจอปัญหา ไม่ใช่เงียบหรือเดา

🔒

4. จำกัด tool access

ให้สิทธิ์เครื่องมือเท่าที่จำเป็นกับงานนั้น

💡 ใช้ตอนไหนถึงคุ้ม

ใช้เมื่อ exploration แยกออกจาก execution ได้ชัด — คือคุณต้องการแค่ "ผลลัพธ์" ไม่ใช่ทุกขั้นตอนระหว่างทาง เหมาะมากเมื่ออยาก กันไม่ให้ main context รก หรืออยากได้ มุมมองใหม่ที่สดและไม่มี bias (เช่น งาน research หรือ code review)

🎯 สรุปใจความสำคัญ

  • Subagent รันใน context window แยก แล้วส่งเฉพาะสรุปกลับ main thread
  • สร้างเป็น markdown + YAML frontmatter ผ่าน /agents (เลือก scope Project/User)
  • ออกแบบให้ดี 4 ข้อ: description ดี, output format, รายงานอุปสรรค, จำกัด tool access
  • ใช้เมื่อแยก exploration ออกจาก execution — ต้องการผลลัพธ์ ไม่ใช่ทุกขั้นตอน และกัน context รก
