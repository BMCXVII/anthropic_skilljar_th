Introduction to Claude Cowork

Claude Cowork คือโหมดบนเดสก์ท็อปที่คุณ ชี้ไปที่โฟลเดอร์ + เชื่อมแอป แล้วให้มันวางแผน ลงมือ และส่งมอบงานทั้งชิ้น — เหมือนมีเพื่อนร่วมงานที่ทำงานหลายขั้นตอนให้จบ

⏱️ 15 บท · ระดับเริ่มต้น 🎯 ต้องใช้ Claude Desktop 📌 สถานะ: ลงเรียนแล้ว

คอร์สนี้พาไล่จาก "Cowork คืออะไรและตั้งค่าอย่างไร" → ปรับแต่งให้เข้ากับงานของคุณ (global instructions/projects/skills/plugins) → ใช้ Claude ทุกที่ที่ทำงาน (Chrome, Microsoft 365) → จนถึงการแชร์และความปลอดภัย

🤝 รู้จัก Claude Cowork

Cowork ต่างจากโหมด Chat ตรงที่มัน ทำงานบนไฟล์จริงและข้ามหลายเครื่องมือ ได้ คุณชี้ให้มันดูโฟลเดอร์หนึ่ง เชื่อมแอปอย่าง Gmail/Slack/Drive แล้วสั่งงาน มันจะวางแผน-ลงมือ-ส่งมอบเป็นชิ้นงานให้ จุดสำคัญด้านความปลอดภัยคือมัน จะถามก่อนเสมอ เมื่อจะ send / delete / share

▶ วิดีโอ: แนะนำ Claude Cowork

ตั้งค่าและมอบงานแรก

ติดตั้ง Claude Desktop จาก claude.com/download แล้วเลือกโหมด Cowork การตั้งค่าที่สำคัญที่สุดคือ "Work in a project" แล้วเลือกโฟลเดอร์ ที่จะให้ Cowork ทำงานด้วย งานที่เหมาะกับ Cowork มี 3 แบบ: หลายขั้นตอน (multi-step), อิงไฟล์จริง, และ ข้ามหลายเครื่องมือ นอกจากนี้ยังตั้ง /schedule ให้ทำงานตามเวลา และใช้ Dispatch สั่งงานจากมือถือได้

เวลามอบงาน Claude จะ ถาม clarify ก่อนเริ่ม ระหว่างทาง steer ปรับทิศได้ และผลลัพธ์ออกมาเป็น artifact (ไฟล์) ให้รีวิวเหมือนรับงานจากเพื่อนร่วมงาน

🧩 ปรับ Cowork ให้เป็นของคุณ — 4 building blocks

📋

Global instructions

คำสั่ง/ความชอบที่ใช้ได้กับทุก session — ตั้งครั้งเดียวมีผลตลอด

📁

Projects

สร้าง memory จากบทสนทนาในโปรเจกต์นั้น ให้บริบทสะสมต่อเนื่อง

📦

Skills

reusable playbook ที่สอนวิธีทำงานแบบที่คุณต้องการ โหลดอัตโนมัติเมื่อ task ตรง

🔌

Plugins

ชุดของ skills + connectors + subagents รอบงานหนึ่ง ไว้ encode วิธีทำงานของทีม (Anthropic มี plugin ตาม role ให้)

🌐 ใช้ Claude ทุกที่ที่ทำงาน

Claude in Chrome เป็นสะพานสำหรับเครื่องมือที่ยังไม่มี connector — อ่านและลงมือบนหน้าเว็บได้ ทำงานคู่กับ Cowork และก็ ถามก่อน action สำคัญ เช่นกัน ส่วน Claude for Microsoft 365 เป็น add-in ใน Word/Excel/PowerPoint/Outlook ที่ carry context ข้ามแอปได้ — แนวคิดคือใช้ M365 ไว้ ขัดเกลา ไฟล์ที่ทำอยู่ ส่วน Cowork ไว้ สร้าง deliverable

🛡️ แชร์และความปลอดภัย

  • ทำงานอย่างปลอดภัย: ใช้โฟลเดอร์ทำงานเฉพาะ (อย่าใช้ Documents/Downloads/Desktop ตรง ๆ), เขียน prompt ให้ไม่กำกวม และรู้จังหวะ "stop and think"
  • Validate ก่อนแชร์: ใช้ eval ทดสอบ skill ก่อนเอาไปแชร์ รัน lightweight eval ผ่าน skill-creator ได้
  • แชร์กับทีม: bundle skills เป็น plugin แล้วแจกทั้งองค์กร (วิธีระดับ Enterprise)

🎯 สรุปใจความสำคัญ

  • Cowork = ชี้โฟลเดอร์ + เชื่อมแอป แล้วให้ทำงาน multi-step บนไฟล์จริงจนจบเป็น artifact
  • ถามก่อนเสมอเมื่อจะ send/delete/share — ปลอดภัยกว่าโดยดีไซน์
  • ปรับแต่งด้วย 4 building blocks: global instructions, projects, skills, plugins
  • ใช้ Claude in Chrome และ Microsoft 365 เพื่อทำงานต่อในเครื่องมือที่ใช้จริง
  • ทำงานปลอดภัยด้วยโฟลเดอร์เฉพาะ, validate skill ก่อนแชร์, bundle เป็น plugin แจกทีม
