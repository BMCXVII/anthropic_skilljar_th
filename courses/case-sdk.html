Case Study: Agent SDK
รัน Claude Code แบบ programmatic

โจทย์: อยากให้ "ตัวช่วยเขียนโค้ด" ทำงานอัตโนมัติในสคริปต์/CI ของเรา ไม่ใช่พิมพ์มือใน terminal — ใช้ Agent SDK ที่ใช้ agent loop เดียวกับ Claude Code

🎯 TypeScript / Python🧩 อิงคอร์ส Claude Code in Action📋 step-by-step
🧭
สถานการณ์ Agent SDK ให้เรารัน Claude Code จากโค้ดของเราเอง ด้วย agent loop เดียวกับ CLI — เหมาะกับงานอัตโนมัติ เช่น สร้าง changelog, ตอบ issue, หรือ batch แก้ไฟล์ แพ็กเกจคือ @anthropic-ai/claude-agent-sdk

1

ติดตั้งแพ็กเกจ

# TypeScript / Node
npm install @anthropic-ai/claude-agent-sdk

# ตั้งค่า API key ผ่าน environment variable
export ANTHROPIC_API_KEY=sk-ant-...
2

เขียนสคริปต์เรียก agent loop

ทำไม: SDK ใช้ loop เดียวกับ CLI (รวบรวม context → ลงมือผ่าน tools → วนซ้ำ) เราแค่ส่ง prompt แล้วรับผลลัพธ์มาใช้ต่อในโปรแกรม

// gen-changelog.ts
import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const msg of query({
  prompt: "อ่าน git log 20 รายการล่าสุด แล้วสร้าง CHANGELOG.md จัดกลุ่มเป็น Features/Fixes",
})) {
  console.log(msg);
}
💡
โครงสร้าง API ที่แน่นอน (ชื่อฟังก์ชัน/ออปชัน) ให้ยึดตามเอกสารทางการของ SDK เวอร์ชันที่ใช้ — แนวคิดหลักคือ "ส่ง prompt → รับสตรีมข้อความ/ผลของ tool กลับมา"
3

รันและเก็บผลไปใช้ต่อ

npx tsx gen-changelog.ts

เพราะมันรันเป็นโค้ด เราจึงเอาไปต่อใน CI/CD ได้ เช่น ให้รันตอนเปิด PR เพื่อสร้าง changelog หรือสรุปการเปลี่ยนแปลงอัตโนมัติ

✅
ได้ความสามารถแบบ Claude Code (แก้ไฟล์/รันคำสั่ง) แต่สั่งจากสคริปต์ ไม่ต้องมีคนนั่งพิมพ์
4

เลือกใช้ให้เหมาะ: SDK vs CLI

  • CLI (claude): งานโต้ตอบรายวัน นั่งทำกับมือ
  • Agent SDK: งานอัตโนมัติ/ทำซ้ำ/ฝังในระบบอื่น — เรียกจากโค้ด

🎯 สิ่งที่ได้จากเคสนี้

  • Agent SDK รัน Claude Code แบบ programmatic ด้วย agent loop เดียวกับ CLI
  • แพ็กเกจ @anthropic-ai/claude-agent-sdk รองรับ TypeScript/Python
  • เหมาะกับงานอัตโนมัติ/CI — ส่ง prompt แล้วรับผลกลับมาใช้ต่อ
  • ยึดเอกสารทางการสำหรับ API ที่แน่นอนของเวอร์ชันที่ใช้
