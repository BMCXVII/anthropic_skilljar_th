Case Study: Claude Code
เพิ่มฟีเจอร์จริงด้วย Explore → Plan → Code → Commit

โจทย์: เว็บแอปของเรายังไม่มี dark mode เราจะให้ Claude Code เพิ่มปุ่มสลับธีมให้ — ทำตามทีละขั้นแบบที่ใช้กับงานจริงได้เลย

🎯 ใช้กับ Claude Code🧩 อิงคอร์ส Claude Code 101📋 step-by-step
🧭
สถานการณ์ คุณมีโปรเจกต์เว็บอยู่แล้วในเครื่อง อยากเพิ่มปุ่ม dark/light toggle ที่จำค่าไว้ได้ — แต่ไม่อยากให้ Claude แก้มั่วทั้งโปรเจกต์ จึงใช้ workflow ที่แยก "วางแผน" ออกจาก "ลงมือ"
1

เปิดโปรเจกต์ + ให้ Claude รู้จักโค้ดเบส

/init จะสร้างไฟล์ CLAUDE.md ที่สรุปบริบทโปรเจกต์ Claude จะอ่านอัตโนมัติทุกครั้ง ทำให้ไม่ต้องอธิบายซ้ำ

# เข้าโฟลเดอร์โปรเจกต์แล้วเริ่ม
claude

# ในเซสชัน Claude Code
/init
✅
ได้ไฟล์ CLAUDE.md ที่บันทึกคำสั่ง build/test และโครงสร้างโปรเจกต์
2

Explore — ให้สำรวจก่อน อย่าเพิ่งให้แก้

ขอให้ Claude อ่านส่วนที่เกี่ยวกับธีม/สไตล์ก่อน เพื่อเข้าใจของเดิม

"อ่านไฟล์ที่เกี่ยวกับการจัดการธีม/สี แล้วสรุปให้ฟังว่าตอนนี้สไตล์ถูกตั้งที่ไหนบ้าง ยังไม่ต้องแก้"
3

Plan — เข้า Plan Mode ขอแผนก่อนลงมือ

Plan Mode ให้ Claude ร่างแผนโดย ยังแก้ไฟล์ไม่ได้ เรารีวิวแผนให้ตรงก่อน ลดการต้องรื้อทีหลัง

# กด Shift+Tab จนเข้าโหมด Plan แล้วพิมพ์
"วางแผนเพิ่มปุ่ม dark/light toggle ที่บันทึกค่าไว้ใน localStorage
และใช้ค่าเดิมตอนเปิดหน้าใหม่ บอกเป็นขั้นตอนว่าจะแก้ไฟล์ไหนบ้าง"
💡
ถ้าแผนไม่ตรง ให้บอกแก้ตรงนี้เลย — ถูกกว่าการให้แก้โค้ดแล้วค่อยรื้อ
4

Code — อนุมัติแล้วให้ลงมือ + รันเทสต์

"แผนโอเค ลงมือตามแผนได้เลย เสร็จแล้วรันเทสต์/เปิดแอปยืนยันว่าใช้งานได้จริง"

กด Esc ได้ทุกเมื่อถ้าอยากหยุดแล้วเปลี่ยนทิศ และถ้า context เริ่มยาวให้ /compact

5

Commit — รีวิวด้วย subagent แล้ว commit

"ใช้ subagent แบบ read-only รีวิวการเปลี่ยนแปลงนี้ก่อน
แล้วถ้าโอเคให้ commit พร้อมเขียน commit message ให้"
✅
ได้ฟีเจอร์ dark mode + เทสต์ผ่าน + commit ที่อธิบายการเปลี่ยนแปลงชัดเจน

📋 Prompt สรุป — คัดลอกไปใช้ได้เลย

⌨️ Master Prompt — เพิ่มฟีเจอร์อย่างปลอดภัย
งาน: เพิ่มปุ่ม dark/light toggle ที่จำค่าใน localStorage

ทำตามขั้นตอนนี้:
1) อ่านไฟล์ที่เกี่ยวกับธีม/สไตล์ แล้วสรุปของเดิมให้ฟัง (ยังไม่แก้)
2) เข้า Plan Mode เสนอแผนแก้ทีละไฟล์ให้ฉันอนุมัติก่อน
3) อนุมัติแล้วค่อยลงมือ + รันเทสต์/เปิดแอปยืนยัน
4) ใช้ subagent read-only รีวิว แล้ว commit พร้อม message

เริ่มจาก /init ถ้ายังไม่มี CLAUDE.md

🎯 สิ่งที่ได้จากเคสนี้

  • ใช้ /init + CLAUDE.md ให้ Claude เข้าใจโปรเจกต์ก่อนเริ่ม
  • แยก Explore/Plan ออกจาก Code — รีวิวแผนก่อนลงมือเสมอ
  • คุม context ด้วย Esc และ /compact
  • รีวิวด้วย subagent read-only ก่อน commit
