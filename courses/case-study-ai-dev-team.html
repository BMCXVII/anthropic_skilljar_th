Building an AI Dev Team
for a Startup Software House

จำลอง "ทีม AI developer" ที่คุยกันจริง พร้อม dashboard — ดูทุก agent ทำงานประสานกันตั้งแต่รับ requirement จนเปิด PR โดยใช้ subagents, Skills จาก Git, MCP, RAG และ Hooks

🎮 จำลองแบบ interactive📊 ระดับสูง🎯 ประกอบทุกฟีเจอร์เข้าด้วยกัน
🧭
วิธีใช้หน้านี้ กด ▶ เริ่มจำลอง เพื่อดูทีม AI คุยกันทำงานทีละขั้น (หรือกด "ก้าวต่อไป" เพื่อเดินเอง) — ดู dashboard อัปเดตความคืบหน้า จากนั้นคัดลอก prompt สรุป ด้านล่างไปลองตั้งทีมจริงใน Claude Code ได้เลย · อิงเอกสารทางการ code.claude.com/docs

🎮 จำลองทีม: รับ Issue #57 → ส่งมอบ PR

📋 Prompt สรุป — คัดลอกไปใช้ได้เลย

วาง prompt นี้ใน Claude Code (หลัง /init) เพื่อตั้งทีม AI dev แบบในภาพจำลองด้านบน:

🧑‍💻 Master Prompt — ตั้งทีม AI Dev
ช่วยตั้งทีม AI developer สำหรับโปรเจกต์นี้ตามนี้:

1) สร้าง subagents ใน .claude/agents/ :
   - architect : read-only (Read, Grep, Glob), model opus — ออกแบบ + วางแผนสถาปัตยกรรม
   - backend   : Read, Edit, Write, Bash — เขียน API/DB ตาม skill coding-standards
   - frontend  : Read, Edit, Write — ทำ UI ตาม design system
   - qa        : read-only — รีวิว bug + ความปลอดภัย ก่อน merge

2) ตั้ง hook PostToolUse (matcher Edit|Write) ให้รัน prettier + tsc อัตโนมัติ
   และ hook PreToolUse (matcher Bash) บล็อกคำสั่งอันตราย (rm -rf, DROP TABLE)

3) ใช้ workflow: explore → plan → code → commit เสมอ
   เมื่อฉันให้ GitHub issue ให้: ดึงรายละเอียด issue ผ่าน MCP →
   architect วางแผน → ฉันอนุมัติ → backend/frontend ลงมือ + เขียนเทสต์ →
   qa รีวิว → /code-review + /verify → เปิด PR ที่ลิงก์ issue

เริ่มจากรัน /init แล้วสรุป "แผนการตั้งทีม" ให้ฉันอนุมัติก่อนลงมือจริง
💡
ปรับชื่อ tool/โมเดล/ขั้นตอนให้ตรงโปรเจกต์ของคุณได้ · ตรวจ syntax ล่าสุดของ subagent frontmatter และ hooks ที่ docs/en/sub-agents และ docs/en/hooks

🧩 เบื้องหลังแต่ละขั้น (อ้างอิงโค้ดจริง)

1

ตั้ง subagent แต่ละบทบาท

ทำไม: แต่ละ subagent มี context window แยก จำกัด tool/model ตามบทบาท Claude หลัก delegate ให้ตาม description

# .claude/agents/architect.md
---
name: architect
description: ออกแบบสถาปัตยกรรม/วางแผนก่อนเขียนโค้ด ใช้เมื่อรับฟีเจอร์ใหม่
tools: Read, Grep, Glob
model: opus
---
คุณคือ Software Architect เสนอโครงสร้างไฟล์ สัญญา API และความเสี่ยง
เป็นแผนขั้นตอนที่ backend/frontend ทำต่อได้
✅
/agents เห็น architect, backend, frontend, qa · สั่ง "ใช้ architect วางแผน..." แล้ว delegate อัตโนมัติ
2

ดึง Skills จาก Git + ผูกกับ agent

ทำไม: เก็บมาตรฐานทีม (coding-standards, api-conventions) เป็น Skill กลางใน Git ให้ทุกคนใช้เหมือนกัน

# clone skills กลางเข้าโปรเจกต์แล้ว commit
git clone https://git.example.com/team/team-skills .claude/skills-src
cp -r .claude/skills-src/coding-standards .claude/skills/
git add .claude/skills && git commit -m "add team skills"

# หรือแจกเป็น plugin ผ่าน marketplace
/plugin marketplace add your-org/claude-plugins
/plugin install team-skills@your-org

ผูกกับ subagent ด้วยฟิลด์ skills: ใน frontmatter (preload เนื้อหาเข้า context). หมายเหตุ: ปัจจุบัน custom commands รวมกับ Skills แล้ว

3

เชื่อม MCP (GitHub + ฐานข้อมูล)

claude mcp add --transport http github https://api.githubcopilot.com/mcp/ \
  --header "Authorization: Bearer $GITHUB_PAT"
claude mcp add db -- npx -y @bytebase/dbhub \
  --dsn "postgresql://readonly:pass@localhost:5432/app"
/mcp   # ดูสถานะ + ยืนยันตัวตน (OAuth)

แชร์ทั้งทีมด้วย project scope ใน .mcp.json (commit ได้) — ให้สิทธิ์ DB แบบ read-only

4

RAG + LLM Wiki ให้ทีมเข้าถึงความรู้

เก็บความรู้ (ADR/สเปก/runbook) ใน docs/ + ทำ Skill ที่ดึงสารบัญเข้า prompt อัตโนมัติด้วย dynamic injection:

---
name: project-wiki
description: คลังความรู้โปรเจกต์ ใช้เมื่อถามเรื่องการตัดสินใจออกแบบ/ระบบเดิม
---
## สารบัญ wiki
!`find docs/wiki -name '*.md' | sed 's|docs/wiki/||'`
อ่านหัวข้อที่เกี่ยวข้องก่อนตอบ และอ้างอิงไฟล์เสมอ

หลัก RAG (chunk → embed → retrieve, + BM25) เรียนเต็มในคอร์ส Building with the Claude API

5

Hooks บังคับมาตรฐานอัตโนมัติ

// .claude/settings.json
{ "hooks": {
  "PostToolUse": [{ "matcher": "Edit|Write",
    "hooks": [{ "type": "command",
      "command": "npx prettier --write \"$CLAUDE_PROJECT_DIR\" && npx tsc --noEmit" }] }],
  "PreToolUse":  [{ "matcher": "Bash",
    "hooks": [{ "type": "command", "command": ".claude/hooks/guard.sh" }] }]
}}
✅
หลัง Claude แก้ไฟล์ โค้ดถูก format + ตรวจ type อัตโนมัติ และคำสั่งอันตรายถูกบล็อก (exit 2)

✅ ฟีเจอร์ที่ใช้ครบใน Case Study นี้

  • Subagents — architect/backend/frontend/qa แยก context + จำกัด tools/model + skills: preload
  • Skills จาก Git — clone เข้า .claude/skills/ หรือ plugin marketplace
  • MCP — GitHub + Postgres, .mcp.json project scope
  • RAG + LLM Wiki — docs ใน repo + wiki skill (dynamic injection)
  • Hooks — PostToolUse (format/typecheck) + PreToolUse (guard)
  • Workflow — explore → plan → code → commit + /code-review + /verify + PR; มนุษย์ approve เสมอ
