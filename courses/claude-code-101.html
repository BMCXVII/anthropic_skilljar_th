Claude Code 101

รู้จัก Claude Code — coding agent ที่ทำงานในเทอร์มินัล/IDE ของคุณจริง ๆ พร้อมจับหัวใจของการทำงานแบบ agentic ผ่าน workflow Explore → Plan → Code → Commit

⏱️ 13 บท · ระดับเริ่มต้น (dev) 🎯 ต้องมี Claude Pro/Max หรือ API key 📌 สถานะ: ลงเรียนแล้ว · คืบหน้า ~92%

คอร์สนี้พาไล่จาก "Claude Code คืออะไร" → ติดตั้งและสั่งงานครั้งแรก → workflow ที่ใช้ทุกวัน → จนถึงการปรับแต่งด้วย CLAUDE.md, Subagents, Skills, MCP และ Hooks. ด้านล่างเรียบเรียงตามลำดับบทเรียนจริง พร้อมวิดีโอประกอบในแต่ละหัวข้อ.

🤖 Claude Code คืออะไร

Claude Code คือ agentic coding tool ที่เข้าใจทั้ง codebase แก้ไฟล์ และรันคำสั่งให้ได้เอง ใช้ได้หลายที่ทั้งในเทอร์มินัล, VS Code, JetBrains, Claude Desktop และบนเว็บ จุดที่ต่างจากการคุยใน Claude.ai คือมัน เข้าถึงไฟล์และเทอร์มินัลของคุณได้โดยตรง จึงไม่ต้องคอย copy-paste โค้ดไปมา

⌨️

Autocomplete แบบเดิม

เติมโค้ดทีละบรรทัดตรงเคอร์เซอร์ — คุณต้องนำทุกขั้นตอนเอง

🤖

Coding Agent

รับเป้าหมายระดับงาน แล้ววางแผน แก้หลายไฟล์ รันคำสั่ง ตรวจผล และวนแก้จนเสร็จ

How Claude Code works — agentic loop

หัวใจคือวงจรที่เรียกว่า agentic loop: ป้อน prompt → Claude รวบรวม context (คุยกับโมเดล ได้ผลเป็นข้อความหรือการเรียก tool) → ลงมือทำ (เช่น อ่าน/แก้ไฟล์ รันคำสั่ง) → วนซ้ำ จนงานเสร็จ พูดง่าย ๆ คือมันทำงานคล้าย developer คนหนึ่งที่ค่อย ๆ สำรวจ ลงมือ แล้วตรวจผลเอง

▶ วิดีโอ: อธิบาย agentic loop ของ Claude Code

🚀 ติดตั้งและ prompt แรก

ติดตั้งได้หลายช่องทางตามระบบของคุณ: macOS / Linux / WSL ใช้ curl (หรือ Homebrew), Windows ใช้ PowerShell / cmd / winget. เสร็จแล้วเข้าโฟลเดอร์โปรเจกต์แล้วพิมพ์ claude เพื่อเริ่ม (ครั้งแรกจะให้เลือกธีม) — ดูคำสั่งติดตั้งล่าสุดได้ที่เอกสารทางการ

# เข้าโฟลเดอร์โปรเจกต์แล้วเริ่ม
claude

# แล้วสั่งงานเป็นภาษาธรรมชาติได้เลย เช่น
"อธิบายโครงสร้างโปรเจกต์นี้ให้หน่อย"
"เพิ่ม endpoint /health ที่คืนค่า status ok แล้วเขียนเทสต์"
⌨️
เลือกโหมดอนุมัติด้วย Shift+Tab สลับระหว่าง Approval (ขออนุญาตทุกครั้งก่อนแก้/รัน — ปลอดภัยกว่า) กับ Auto-Accept (ปล่อยให้ทำต่อเนื่อง)

🧭 Workflow ที่ใช้ทุกวัน

Explore → Plan → Code → Commit

นี่คือ workflow ที่คอร์สเน้นว่าสำคัญที่สุด แนวคิดคือ แยก "วางแผน" ออกจาก "ลงมือ" ก่อน ให้ Claude อ่านไฟล์ที่เกี่ยวข้องและเข้าใจของเดิมเสียก่อน (Explore) แล้วใช้ Plan Mode ให้มันร่างแผนโดยที่ยังแก้ไฟล์ไม่ได้ (Plan) เมื่อแผนตรงใจค่อยให้ลงมือ (Code) แล้วจบด้วยการ commit การทำแบบนี้ช่วยลดการต้องรื้อแก้ทีหลังได้มาก

🔍

1. Explore

ให้ Claude อ่าน/ทำความเข้าใจโค้ดเบสที่เกี่ยวข้องก่อน อย่าเพิ่งให้แก้

📝

2. Plan

ใช้ Plan Mode (Shift+Tab) ให้วางแผนก่อนลงมือ เพื่อรีวิวแนวทางให้ตรง

⌨️

3. Code

ลงมือแก้ตามแผน รันเทสต์/คำสั่งเพื่อยืนยันว่าทำงานจริง

✅

4. Commit

สรุปการเปลี่ยนแปลงแล้ว commit (ให้ช่วยเขียน commit message ได้)

▶ วิดีโอ: workflow หลักของ Claude Code + Plan Mode

Context management — คุมหน่วยความจำการสนทนา

มองว่า context window คือ working memory ของ Claude ทุก prompt, ไฟล์ที่อ่าน, และผลของ tool ล้วนกินพื้นที่ เมื่อใกล้เต็ม Claude จะ compact ให้อัตโนมัติ (สรุปแล้วตัดของเก่าทิ้ง) หรือคุณสั่งเองได้ด้วย /compact และถ้าจะเริ่มงานใหม่ที่ไม่เกี่ยวกับงานเดิมก็ /clear ล้างให้สะอาด

▶ วิดีโอ: context window และ /compact

Code review ก่อน push

ก่อนจะ push เป็น PR ใช้ subagent ที่เป็น read-only มารีวิวการเปลี่ยนแปลงด้วย context ที่สดและไม่มี bias ติดจากตอนเขียน นอกจากนี้ยังมี skill /commit-push-pr ที่ทำ commit + push + เปิด PR ให้ในขั้นตอนเดียว

⚙️ ปรับแต่ง Claude Code ให้เข้ากับงานคุณ

CLAUDE.md — หน่วยความจำถาวรของโปรเจกต์

ไฟล์ Markdown ชื่อ CLAUDE.md ที่ราก repo จะถูก อ่านอัตโนมัติทุก session มองว่าเป็น "onboarding script" ของโค้ดเบส — ใส่คำสั่ง build/test, สไตล์โค้ด, และข้อตกลงของทีมไว้ Claude จะทำงานตรงแนวทางโดยไม่ต้องบอกซ้ำ (สร้างได้ด้วย /init)

▶ วิดีโอ: CLAUDE.md = persistent project memory

Subagents — ผู้ช่วยใน context แยก

เมื่องานหนึ่งจะกิน context เยอะ (เช่น สำรวจโค้ดเบสทั้งก้อน) ให้ delegate งานนั้นไปให้ subagent ที่รันใน context window ของตัวเอง แล้วส่ง เฉพาะสรุป กลับมาที่ main thread — ช่วยให้บริบทหลักไม่รก

▶ วิดีโอ: subagent รันใน context แยก

Skills

คอร์สแตะ Skills แบบสั้น ๆ แล้วชี้ให้ไปเรียนต่อในคอร์สเฉพาะ — Skills คือชุดความรู้/วิธีทำงานที่ Claude โหลดมาใช้เมื่อจำเป็น ดูคอร์ส Introduction to Agent Skills →

MCP

MCP (Model Context Protocol) เป็น open standard ที่เชื่อม Claude Code เข้ากับ tools/data ภายนอก เช่น ฐานข้อมูล แอป หรือ repo (ตัวอย่างคือ Linear) ทำให้ Claude ทำงานกับระบบจริงขององค์กรได้

▶ วิดีโอ: MCP เชื่อม Claude Code กับ tools/data ภายนอก

Hooks

Hooks คือคำสั่งที่ตั้งให้รันในจุดต่าง ๆ ของ lifecycle จุดเด่นคือ deterministic — รันแน่นอนทุกครั้ง (ต่างจากการพึ่งให้โมเดล "จำ" ทำเอง) ใช้ทำงานอย่าง auto-format, logging หรือบล็อกงานอันตราย ตั้งค่าในไฟล์ settings.json

▶ วิดีโอ: hooks = deterministic รันแน่นอนทุกครั้ง

⌨️ คำสั่ง/คีย์ที่ใช้บ่อย

🎯 สรุปใจความสำคัญ

  • Claude Code เป็น coding agent ที่ลงมือทำจริง (อ่าน/แก้ไฟล์ รันคำสั่ง) ไม่ใช่แค่ autocomplete
  • ทำงานเป็น agentic loop: รวบรวม context → ลงมือ → วนซ้ำ
  • Workflow ทองคำคือ Explore → Plan → Code → Commit — แยกวางแผนออกจากลงมือ
  • คุม context ด้วย /compact & /clear; ให้บริบทถาวรด้วย CLAUDE.md (/init)
  • ปรับแต่งต่อยอดด้วย Subagents (context แยก), Skills, MCP (เชื่อมภายนอก) และ Hooks (รันแน่นอนทุกครั้ง)
