Case Study: MCP
ให้ Claude ตัดต่อวิดีโอผ่าน DaVinci Resolve

โจทย์: มีคลิปดิบหลายไฟล์ อยากให้ Claude ตัดต่อ ใส่ subtitle ปรับสี export ได้โดยตรง — ทำได้ด้วยการต่อ Claude Code เข้ากับ DaVinci Resolve ผ่าน MCP server

🎯 ใช้กับ Claude Code🧩 อิงคอร์ส Introduction to MCP📋 step-by-step
🧭
สถานการณ์ MCP server คือสะพานที่ expose ความสามารถของโปรแกรมภายนอก (เช่น DaVinci Resolve) ให้ Claude เรียกใช้ได้ผ่าน tools มาตรฐาน — ผู้ใช้ทั่วไป ไม่ต้องสร้าง server เอง แค่เพิ่มเข้า Claude Code แล้วสั่งงานได้เลย
1

ทำความเข้าใจ: MCP ในมุมผู้ใช้

ก่อนลงมือ ดูว่า pieces ทั้งหมดวางตัวยังไง:

# ภาพรวม flow
คุณพิม prompt
    ↓
Claude Code
    ↓ (MCP protocol)
DaVinci Resolve MCP Server   ← สะพาน
    ↓ (Lua scripting API)
DaVinci Resolve              ← โปรแกรมทำงานจริง

Claude ไม่ได้ "เห็น" วิดีโอโดยตรง แต่สั่งงานผ่าน tools ที่ MCP server expose ออกมา เช่น import_clip, add_subtitle, export_timeline

💡
MCP server สำหรับโปรแกรมยอดนิยมมักมีคนทำไว้แล้ว — ค้นหา [ชื่อโปรแกรม] MCP server ใน GitHub ก่อนสร้างเอง
2

เตรียม DaVinci Resolve + MCP Server

ต้องมีสองอย่างพร้อมกัน:

🎬

DaVinci Resolve

ติดตั้งและเปิดค้างไว้ก่อนเสมอ เพราะ MCP server จะเชื่อมผ่าน scripting API ที่รันอยู่

ใช้ได้ทั้ง Free และ Studio edition

🔌

DaVinci Resolve MCP Server

ติดตั้ง package ที่ทำหน้าที่เป็นสะพาน:

pip install davinci-resolve-mcp

ใน DaVinci Resolve เปิดใช้ scripting API ที่ Preferences → System → General → Enable external scripting using local network

3

เพิ่ม MCP server เข้า Claude Code

บอก Claude Code ว่าจะใช้ MCP server ตัวไหน เลือกได้ 2 วิธี:

วิธีที่ 1 — ผ่าน command line (ใช้ครั้งเดียว)

claude mcp add davinci-resolve -- python -m davinci_resolve_mcp

วิธีที่ 2 — เพิ่มใน .mcp.json (แชร์ทีมได้)

# .mcp.json ในโฟลเดอร์โปรเจกต์
{
  "mcpServers": {
    "davinci-resolve": {
      "command": "python",
      "args": ["-m", "davinci_resolve_mcp"]
    }
  }
}
💡
commit .mcp.json เข้า repo ได้เลย ทีมทุกคนจะได้ใช้ MCP server เดียวกันโดยไม่ต้อง add ซ้ำ
4

ตรวจว่าต่อติดและเห็น tools

เปิด Claude Code แล้วพิม:

/mcp

ถ้าต่อสำเร็จจะเห็น davinci-resolve ใน list พร้อม tools ที่ expose ออกมา เช่น:

# ตัวอย่าง tools ที่เห็น
davinci-resolve:
  - import_media      (นำเข้าไฟล์วิดีโอ)
  - create_timeline   (สร้าง timeline ใหม่)
  - add_clip          (วางคลิปบน timeline)
  - add_subtitle      (ใส่ subtitle)
  - apply_color_grade (ปรับสี/LUT)
  - export_timeline   (render และ export)
✅
เห็น tools ครบ = พร้อมให้ Claude สั่ง DaVinci Resolve ได้แล้ว
5

สั่ง Claude ตัดต่อวิดีโอ

สั่งเป็นเป้าหมายระดับงาน Claude จะวางแผนและเรียก tools ให้เอง:

🎬 ตัดต่อพื้นฐาน
ใน DaVinci Resolve:
1. นำเข้าไฟล์คลิปทั้งหมดใน /Videos/raw/
2. สร้าง timeline ชื่อ "highlight-reel" 1920x1080 24fps
3. เรียงคลิปตามลำดับชื่อไฟล์ ตัดให้เหลือเฉพาะช่วง 0:10–0:40 ของแต่ละคลิป
4. export เป็น MP4 H.264 ไว้ที่ /Videos/output/
📝 ใส่ subtitle จากไฟล์ script
อ่านไฟล์ script.txt แล้วใส่ subtitle ลงใน timeline "highlight-reel"
จัดเวลาให้ตรงกับช่วงที่คลิปแต่ละตัวเล่น
ใช้ฟอนต์ Sarabun ขนาด 36 สีขาว outline ดำ
🎨 ปรับสีทั้ง timeline
ปรับสีคลิปทุกตัวใน timeline ให้โทนสีอุ่น
เพิ่ม contrast เล็กน้อย แล้ว apply LUT "Cinematic Warm" ถ้ามีใน library
6

MCP server ของโปรแกรมอื่น ๆ

pattern เดียวกันนี้ใช้ได้กับโปรแกรมใดก็ได้ที่มี MCP server:

โปรแกรมMCP Serverทำอะไรได้
Blenderblender-mcpสร้าง 3D object ปรับ material render scene
Figmafigma-mcpอ่าน/สร้าง component ดึง design token
Premiere Proผ่าน ExtendScript + MCP wrapperตัดต่อ ใส่ effect export
ffmpegffmpeg-mcptranscode trim merge ไม่ต้องเปิด GUI
⚠️
MCP server จากบุคคลที่สาม ตรวจ source code ก่อนรัน เพราะ server ทำงานในเครื่องและเข้าถึงโปรแกรมของเราได้

🎯 สิ่งที่ได้จากเคสนี้

  • MCP server = สะพาน Claude ↔ โปรแกรมภายนอก ไม่ต้องสร้าง integration เอง
  • เพิ่มด้วย claude mcp add หรือ .mcp.json (commit แชร์ทีมได้)
  • สั่งเป็นเป้าหมาย Claude จะวางแผนและเรียก tools ให้เอง
  • pattern เดียวกันใช้ได้กับ Blender, Figma, ffmpeg และโปรแกรมอื่นที่มี MCP server
