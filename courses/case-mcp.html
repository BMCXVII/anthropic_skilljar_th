Case Study: MCP
ต่อ Claude เข้ากับข้อมูล/เครื่องมือของเราเอง

โจทย์: อยากให้ Claude เข้าถึง "ฐานความรู้ภายใน" ของทีม (เช่น คลังบทความ) — สร้าง MCP server เล็ก ๆ แล้วต่อเข้ากับ Claude Code

🎯 Python SDK🧩 อิงคอร์ส Introduction to MCP📋 step-by-step
🧭
สถานการณ์ MCP เป็น communication layer มาตรฐานที่ทำให้ต่อ tools/data ภายนอกได้โดยไม่ต้องเขียน integration เองทุกครั้ง เราจะสร้าง server ที่ expose tool ค้นบทความ และ resource ดึงเนื้อหา
1

ตั้งโปรเจกต์ + ติดตั้ง MCP SDK

uv init my-mcp-server && cd my-mcp-server
uv add "mcp[cli]"
2

นิยาม tool ด้วย decorator

ทำไม: Python SDK ใช้ decorator ทำให้นิยาม tool ได้สั้น ๆ — Claude จะเห็นชื่อ/คำอธิบายแล้วเรียกใช้เมื่อเหมาะ

# server.py
from mcp.server.fastmcp import FastMCP
mcp = FastMCP("kb")

@mcp.tool()
def search_articles(keyword: str) -> list[str]:
    """ค้นบทความในคลังความรู้จาก keyword"""
    return kb_search(keyword)   # ฟังก์ชันจริงของเรา

if __name__ == "__main__":
    mcp.run()
3

เพิ่ม resource (ข้อมูลแบบอ่าน)

resource เหมือน GET handler — expose ข้อมูลให้ใส่เข้า prompt ได้ตรง ๆ

@mcp.resource("article://{id}")
def get_article(id: str) -> str:
    """ดึงเนื้อหาบทความตาม id"""
    return kb_get(id)
4

debug ด้วย server inspector

ก่อนต่อกับ Claude จริง ใช้ inspector ตรวจว่า tool/resource ทำงานถูกต้องแบบ real-time

uv run mcp dev server.py
5

ต่อเข้ากับ Claude Code

claude mcp add kb -- uv run server.py
/mcp   # ดูสถานะว่าต่อติดและเห็น tools/resources
✅
ถามว่า "หาบทความเรื่อง onboarding ให้หน่อย" แล้ว Claude เรียก search_articles ผ่าน MCP server ของเราได้เลย
💡
แชร์ทั้งทีมด้วย project scope ใน .mcp.json (commit ได้); สำหรับ server ระยะไกลใช้ StreamableHTTP transport · syntax ล่าสุดที่ modelcontextprotocol.io

🎯 สิ่งที่ได้จากเคสนี้

  • MCP server expose ได้ทั้ง tools (ฟังก์ชัน) และ resources (ข้อมูลแบบอ่าน)
  • Python SDK + decorator ทำให้นิยามได้สั้น
  • debug ด้วย inspector ก่อน แล้วต่อด้วย claude mcp add
  • แชร์ทีมผ่าน .mcp.json (project scope)
