Case Study: Skill
สร้าง playbook "เขียน PR description" ใช้ซ้ำทั้งทีม

โจทย์: ทุกคนเขียน PR description คนละสไตล์ อยากให้ Claude เขียนให้ รูปแบบเดียวกัน ทุกครั้ง — แก้ด้วยการทำเป็น Skill

🎯 ใช้กับ Claude Code / Cowork🧩 อิงคอร์ส Introduction to Agent Skills📋 step-by-step
🧭
สถานการณ์ Skill คือโฟลเดอร์ + ไฟล์ SKILL.md ที่สอนวิธีทำงานเฉพาะ Claude จะ หยิบมาใช้เองเมื่อ task ตรง ผ่าน semantic matching เราจะทำ skill ที่ทำให้ PR description ออกมาเป็นรูปแบบเดียวกันเสมอ
1

สร้างโฟลเดอร์ skill + ไฟล์ SKILL.md

name + description ใน frontmatter คือสิ่งที่ Claude ใช้ "match" ว่าเมื่อไหร่ควรหยิบ skill นี้ — เขียน description ให้ชัดและตรงงาน

# project: .claude/skills/pr-writer/SKILL.md
---
name: pr-writer
description: ใช้เมื่อผู้ใช้ขอเขียน/สรุป PR description จาก diff หรือ commit
---
# วิธีเขียน PR description ของทีมเรา
เขียนตามหัวข้อนี้เสมอ:
1. Summary — สรุปสั้นว่าทำอะไร/ทำไม
2. Changes — bullet สิ่งที่เปลี่ยน
3. Testing — ทดสอบอะไรไปบ้าง
4. Risk/Notes — ผลกระทบหรือข้อควรระวัง
ภาษากระชับ ใช้ present tense
2

เลือกที่เก็บ: personal หรือ project

เก็บได้ 2 ที่ — personal (~/.claude/skills) ใช้ส่วนตัวทุกโปรเจกต์ หรือ project (.claude/skills) ผูกกับ repo เพื่อแชร์ทีม เคสนี้เลือก project แล้ว restart เพื่อให้ตรวจเจอ

3

ลองใช้งาน — ให้ skill trigger เอง

ไม่ต้องสั่งชื่อ skill ตรง ๆ แค่ขอสิ่งที่ description ครอบคลุม Claude จะ ขอ confirmation ก่อนโหลดเต็ม แล้วทำตาม playbook

"ช่วยเขียน PR description จาก diff ของ branch นี้ให้หน่อย"
✅
ได้ PR description ตามหัวข้อ Summary / Changes / Testing / Risk ทุกครั้ง
4

จัดระเบียบ + แชร์ทีมผ่าน Git

ใช้ progressive disclosure: เก็บ SKILL.md ให้สั้น (< 500 บรรทัด) แล้ว link ไปไฟล์ตัวอย่าง/สคริปต์เพิ่ม จากนั้น commit เข้า repo ให้ทุกคนได้ skill เดียวกัน

git add .claude/skills/pr-writer && git commit -m "add pr-writer skill"
5

แก้ปัญหาเมื่อ skill ไม่ทำงาน

  • ไม่ trigger → ปรับ description ให้ตรงกับคำที่ผู้ใช้พิมพ์จริง
  • ไม่โหลด → ตรวจว่า SKILL.md โครงสร้างถูก (frontmatter ครบ)
  • หยิบผิดตัว → description ของหลาย skill คล้ายกันเกินไป ให้เขียนให้ต่างชัด

📋 เทมเพลต SKILL.md — คัดลอกไปปรับใช้

📦 SKILL.md เริ่มต้น
---
name: ชื่อ-skill-แบบ-kebab-case
description: ใช้เมื่อ... (ระบุสถานการณ์/คำที่ผู้ใช้จะพิมพ์ให้ชัด)
---
# หัวข้อ skill
อธิบายขั้นตอน/กฎที่อยากให้ Claude ทำตาม เป็น bullet สั้น ๆ
(ถ้ายาว ให้แยกไฟล์แล้ว link ถึง — progressive disclosure)

🎯 สิ่งที่ได้จากเคสนี้

  • Skill = โฟลเดอร์ + SKILL.md; description คือกุญแจให้ trigger ถูก
  • เก็บที่ project (.claude/skills) เพื่อแชร์ทีมผ่าน Git
  • ใช้ progressive disclosure ให้ SKILL.md กระชับ
  • แก้ปัญหา trigger/โหลด/หยิบผิด ที่ description เป็นหลัก
