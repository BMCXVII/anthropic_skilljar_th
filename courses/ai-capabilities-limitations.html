AI Capabilities & Limitations

เข้าใจว่าโมเดลทำงานอย่างไรผ่าน 4 properties — แต่ละอย่างมีโซนที่ "เก่ง" และโซนที่ "พลาด" ชัดเจน ทำให้เรารู้ว่าเมื่อไหร่ควรเชื่อและเมื่อไหร่ต้องตรวจ

⏱️ 14 บท · เริ่มต้น–กลาง 🎯 เชื่อมโยงกับ 4D Framework 📌 สถานะ: ลงเรียนแล้ว · คืบหน้า ~28%

แนวคิดหลัก: แทนที่จะท่องจำว่า "AI ทำอะไรได้/ไม่ได้" ให้เข้าใจ 4 คุณสมบัติพื้นฐานของโมเดล แล้วจะเดาได้เองว่างานแบบไหนเสี่ยงพลาด — และเพราะ property เหล่านี้คงที่แม้โมเดลจะพัฒนาขึ้น ความเข้าใจนี้จึง durable

🧭 เริ่มต้น: AI คืออะไรและได้นิสัยมาจากไหน

Generative AI คือ AI ที่สร้างเนื้อหาใหม่ ความเก่ง/อ่อนของมันวางตัวเป็น continuum (ไล่ระดับ) ตาม 4 แกน ไม่ใช่ "ได้/ไม่ได้" แบบขาวดำ เป้าหมายคือสร้าง calibrated trust — เชื่อให้พอดีกับความน่าเชื่อจริง โมเดลได้ "นิสัย" มาจาก pretraining (ฝึกเป็นตัวเติมเอกสาร) + fine-tuning (ปรับให้เป็นผู้ช่วย) ซึ่งทิ้งร่องรอยบางอย่างไว้ เช่น sycophancy (เออออตาม) หรือ verbosity (พูดยืดยาว)

▶ ภาพรวม 4 properties ของโมเดล

▶ Generative AI + 4 แกน continuum

▶ pretraining + fine-tuning

🔢 Property 1 — Next Token Prediction

โมเดลเขียนทีละคำตาม pattern ที่เรียนมา ผลข้างเคียงคือ fabrication (แต่งขึ้น) มักกระจุกที่ ชื่อ/วันที่/สถิติ/citation/URL — ยิ่งเป็นข้อมูลเฉพาะเจาะจง ยิ่งต้อง verify นี่คือรากฐานของ Discernment

📚 Property 2 — Knowledge

ความรู้ของโมเดลถูก "แช่แข็ง" ไว้ที่ knowledge cutoff โซนที่เชื่อได้: เรื่องที่พบบ่อย/มีมานาน/สม่ำเสมอ; โซนที่ต้องระวัง: เรื่องหายาก/เกิดหลัง cutoff/ยังเป็นที่ถกเถียง

▶ วิดีโอ: ความรู้ frozen ที่ knowledge cutoff

🧠 Property 3 — Working Memory

context window มีขนาดจำกัด และเวลาพลาดมักเป็นแบบ "หน้าผา" (cliff) คือตัดข้อมูลเงียบ ๆ (silent truncation) โดยไม่เตือน บทเรียนสำคัญ: อย่าฝังข้อมูลสำคัญไว้กลางข้อความที่ยาวมาก

▶ วิดีโอ: context window มีขีดจำกัด (cliff / silent truncation)

🎛️ Property 4 — Steerability

โมเดลทำตามคำสั่งผ่านกลไก Next Token Prediction เช่นกัน โซนที่ทำได้ดี: คำสั่ง สั้น/รูปธรรม/ตรวจสอบได้; จุดที่พลาด: reasoning drift (ค่อย ๆ ออกนอกลู่) และ letter-over-spirit (ทำตามตัวอักษรแต่ผิดเจตนา)

▶ วิดีโอ: ทำตามคำสั่งผ่าน Next Token Prediction

⚡ เมื่อ properties ชนกัน

ความผิดพลาดจริง ๆ มักไม่ได้มาจาก property เดียว แต่มาจาก 2 อย่างขัดกัน เช่น Next Token Prediction + Knowledge → hallucination, หรือ Working Memory + Steerability → drift ออกนอกคำสั่ง

▶ failure จาก 2 properties โต้กัน

▶ สรุป mental model + เชื่อม 4D

🎯 สรุปใจความสำคัญ

  • เข้าใจโมเดลผ่าน 4 properties: Next Token Prediction, Knowledge, Working Memory, Steerability
  • แต่ละ property เป็น continuum — มีโซน capability และ limitation ชัดเจน
  • Fabrication กระจุกที่ชื่อ/วันที่/สถิติ/citation/URL → ต้อง verify (รากฐานของ Discernment)
  • ความรู้ frozen ที่ cutoff; context window พลาดแบบ cliff (silent truncation)
  • failure จริงมักเกิดจาก 2 properties ชนกัน — mental model นี้เชื่อมตรงกับ 4D
