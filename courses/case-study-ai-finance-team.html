Building an AI Finance
Assistant Team

จำลองทีมผู้ช่วย AI การเงินที่คุยกันจริง พร้อม dashboard — รวบรวมราคา/งบ/ข่าวด้วย Web Search API, สรุปตามกรอบ และออกรายงานพร้อมความเสี่ยง โดยมนุษย์ตัดสินใจขั้นสุดท้าย

🎮 จำลองแบบ interactive📊 ระดับสูง🎯 เน้นออกแบบระบบ + ใช้ Claude

⚠️ ข้อจำกัดความรับผิดชอบ — ไม่ใช่คำแนะนำการลงทุน (Not Financial Advice)

เนื้อหานี้สอน วิธีออกแบบระบบและใช้งาน Claude เพื่อรวบรวม/สรุป/วิเคราะห์ข้อมูลเท่านั้น ไม่ใช่คำแนะนำให้ซื้อ/ขาย/ถือสินทรัพย์ใดๆ

การลงทุนมีความเสี่ยง · ผลในอดีตไม่รับประกันผลในอนาคต · AI อาจให้ข้อมูลผิดพลาดได้ โปรดปรึกษาผู้เชี่ยวชาญด้านการเงินที่มีใบอนุญาตก่อนตัดสินใจลงทุนจริง

🧭
วิธีใช้หน้านี้ กด ▶ เริ่มจำลอง ดูทีมการเงินคุยกันรวบรวม-วิเคราะห์ข้อมูลทีละขั้น แล้วคัดลอก prompt สรุป ไปลองใช้ได้เลย

🎮 จำลองทีม: รวบรวม–วิเคราะห์ข้อมูลหุ้น (TICKER)

📋 Prompt สรุป — คัดลอกไปใช้ได้เลย

📈 Master Prompt — ตั้งทีมผู้ช่วยข้อมูลการเงิน
ช่วยตั้งทีมผู้ช่วย "รวบรวม/วิเคราะห์ข้อมูลการเงิน" (ย้ำ: ไม่ใช่คำแนะนำการลงทุน):

1) เพิ่มหลักการใน CLAUDE.md:
   - หน้าที่ = รวบรวม สรุป วิเคราะห์ข้อมูลเท่านั้น
   - ห้ามให้คำแนะนำซื้อ/ขาย หรือทำนายราคาแบบชี้นำ
   - อ้างอิงแหล่งที่มาเสมอ + ระบุความไม่แน่นอน/ความเสี่ยง
   - แนบ disclaimer "ไม่ใช่คำแนะนำการลงทุน" ทุกรายงาน

2) สร้าง subagents ใน .claude/agents/ :
   - data-collector     : ดึงราคา/งบ (อ่านอย่างเดียว)
   - news-aggregator    : WebSearch, WebFetch — สรุปข่าว + อ้างอิง URL
   - fundamental-analyst: สรุปงบ/อัตราส่วนตามกรอบ (ไม่ชี้นำ)
   - risk-reporter      : read-only — เรียบเรียงรายงาน + ความเสี่ยง + disclaimer

3) ตั้ง hook PreToolUse บันทึก audit log ทุกครั้งที่เรียก market-data/web search

4) เมื่อฉันให้ ticker: ดึงราคา/งบ + ค้นข่าว (มี citation) →
   สรุปพื้นฐานตามกรอบ → ออกรายงานพร้อม "ความเสี่ยง" + disclaimer

ย้ำ: ผลลัพธ์เพื่อการศึกษา ฉัน (มนุษย์) เป็นผู้ตัดสินใจเอง

🧩 เบื้องหลังแต่ละขั้น (อ้างอิงโค้ดจริง)

1

subagent ตามหน้าที่ (least privilege)

# .claude/agents/news-aggregator.md
---
name: news-aggregator
description: รวบรวม/สรุปข่าวตลาดล่าสุด ใช้เมื่อผู้ใช้ขอ "ข่าวล่าสุด"
tools: Read, WebSearch, WebFetch
model: sonnet
---
สรุปเฉพาะข้อเท็จจริงจากแหล่งน่าเชื่อถือ อ้างอิง URL ทุกครั้ง
แยก "ข้อเท็จจริง" จาก "ความเห็น" — ห้ามสรุปเป็นคำแนะนำซื้อ/ขาย
2

Web Search API — ดึงข่าว/ราคาแบบมี citation

web search เป็น server tool (Anthropic รันให้) คืนคำตอบพร้อมแหล่งอ้างอิง เหมาะกับข้อมูลเกินวันตัดความรู้ของโมเดล

import anthropic
client = anthropic.Anthropic()
response = client.messages.create(
    model="claude-opus-4-8", max_tokens=1024,
    messages=[{"role": "user",
        "content": "สรุปข่าวล่าสุดที่กระทบราคาน้ำมันสัปดาห์นี้ พร้อมแหล่งอ้างอิง"}],
    tools=[{"type": "web_search_20250305", "name": "web_search", "max_uses": 5}],
)
🔎
มีรุ่นใหม่ web_search_20260209 (dynamic filtering) ด้วย — ตรวจ version string + โมเดลที่รองรับกับ docs ล่าสุด · ต้องเปิดใช้งานใน Claude Console ก่อน
3

Hooks audit log เพื่อ compliance

// .claude/settings.json
{ "hooks": { "PreToolUse": [{
  "matcher": "mcp__market-data__.*|WebSearch",
  "hooks": [{ "type": "command", "command": ".claude/hooks/audit-log.sh" }]
}]}}
✅
ทุกการเรียก market-data/web search ถูกบันทึกเวลา+ชื่อ tool อัตโนมัติ — ตรวจสอบย้อนหลังได้

⚠️ ย้ำอีกครั้ง

ผลลัพธ์ทั้งหมดเป็นการรวบรวม/วิเคราะห์ข้อมูลเพื่อการศึกษา ไม่ใช่คำแนะนำการลงทุน — ตรวจสอบกับแหล่งทางการ และปรึกษาที่ปรึกษาการเงินที่มีใบอนุญาตก่อนตัดสินใจจริง

✅ ฟีเจอร์ที่ใช้ครบใน Case Study นี้

  • Subagents — data-collector / analyst / news / risk-reporter แยก context + least privilege
  • API Web Search — server tool พร้อม citation (web_search_20250305/20260209)
  • MCP — แหล่งข้อมูลตลาด (read-only + ตรวจ ToS)
  • Hooks — PreToolUse audit log เพื่อ compliance
  • ความรับผิดชอบ — หลักการใน CLAUDE.md + disclaimer ทุกรายงาน + มนุษย์ตัดสินใจ
