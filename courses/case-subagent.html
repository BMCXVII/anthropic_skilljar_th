Case Study: Subagent
แยกงาน "สำรวจ" ออกจาก "ลงมือ"

โจทย์: ก่อนแก้บั๊กในโค้ดเบสใหญ่ ต้องสำรวจหลายไฟล์ — แต่ไม่อยากให้บริบทหลักรกจนหลงประเด็น ใช้ subagent ที่รันใน context แยกแล้วส่งเฉพาะสรุปกลับ

🎯 ใช้กับ Claude Code🧩 อิงคอร์ส Introduction to Subagents📋 step-by-step
🧭
สถานการณ์ subagent คือผู้ช่วยที่รันใน context window แยก ขั้นตอนกลาง ๆ ไม่มารกใน main thread แล้วส่งกลับมาแค่ สรุป — เหมาะมากเมื่อ "ต้องการผลลัพธ์ ไม่ใช่ทุกขั้นตอน"
1

ตัดสินใจว่าควรใช้ subagent ไหม

ใช้เมื่อ exploration แยกจาก execution ได้ชัด เช่น สำรวจโค้ดเบส, รีวิวโค้ด, หรืองาน research ที่อยากได้แค่บทสรุป + อยากกัน main context ไม่ให้รก

2

สร้าง subagent ด้วย /agents

ทำผ่านเมนูง่ายสุด — เลือก scope แล้วให้ Claude ช่วย generate config (เป็น markdown + YAML frontmatter)

/agents
# → Create new agent → เลือก Project หรือ User → ให้ Claude generate
3

ออกแบบให้ดี 4 ข้อ

# .claude/agents/explorer.md
---
name: explorer
description: ใช้สำรวจโค้ดเบสเพื่อหาจุดที่เกี่ยวกับบั๊ก แล้วสรุปกลับ
tools: Read, Grep, Glob          # จำกัด tool ให้ read-only
---
คุณคือผู้สำรวจโค้ด หา root cause ที่เป็นไปได้
ส่งกลับเป็น: ไฟล์/บรรทัดที่เกี่ยว, สมมติฐาน, ขั้นตอนถัดไป
ถ้าติดอุปสรรคให้บอกชัด ไม่ต้องเดา

หลักออกแบบ: (1) description ดี (2) กำหนด output format (3) ให้รายงานอุปสรรค (4) จำกัด tool access

4

ใช้งาน — delegate แล้วรับสรุป

"ใช้ explorer หาสาเหตุที่หน้า checkout คำนวณภาษีผิด
แล้วสรุปจุดที่ต้องแก้กลับมา"
✅
main thread ได้สรุป "ไฟล์/บรรทัด + สมมติฐาน + ขั้นต่อไป" โดยไม่ถูกท่วมด้วยรายละเอียดการไล่อ่านทุกไฟล์
5

ต่อยอด: ทีมเล็ก ๆ ของ subagents

เพิ่ม subagent อื่นที่ทำหน้าที่ต่างกัน เช่น reviewer (read-only รีวิวก่อน push) เพื่อได้ มุมมองสดที่ไม่มี bias ติดจากตอนเขียน

💡
ถ้าจะให้ subagent ใช้ Skill ด้วย ต้อง list skills ใน frontmatter ของ subagent · syntax ล่าสุดที่ code.claude.com/docs/en/sub-agents

🎯 สิ่งที่ได้จากเคสนี้

  • subagent รันใน context แยก ส่งเฉพาะสรุปกลับ — กัน main context รก
  • สร้างผ่าน /agents (markdown + YAML frontmatter)
  • ออกแบบดี 4 ข้อ: description, output format, รายงานอุปสรรค, จำกัด tool
  • ใช้เมื่อแยก exploration ออกจาก execution หรืออยากได้มุมมองใหม่
