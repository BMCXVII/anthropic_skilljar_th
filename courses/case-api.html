Case Study: Claude API
แชตบอตที่จำบทสนทนา + เรียก tool ได้

โจทย์: ทำผู้ช่วยตอบคำถามที่ คุยต่อเนื่องได้ และดึงข้อมูลจริง (เช่น เช็คสภาพอากาศ) ผ่าน tool use — สร้างด้วย Claude API ทีละขั้น

🎯 ระดับ developer🧩 อิงคอร์ส Building with the Claude API📋 step-by-step
🧭
สถานการณ์ สองหัวใจที่ต้องเข้าใจ: (1) API เป็น stateless — ต้องเก็บประวัติ messages เองแล้วส่งทั้งชุดทุกครั้ง (2) tool use ให้ Claude เรียกฟังก์ชันของเราเพื่อดึงข้อมูลนอก training data
1

เตรียม API key + ติดตั้ง SDK

# สร้าง key ที่ console.anthropic.com
export ANTHROPIC_API_KEY=sk-ant-...
pip install anthropic
2

บทสนทนาต่อเนื่อง — เก็บ messages เอง

เพราะ stateless เราต้องสะสม list ของ messages แล้วส่งไปทั้งหมดในแต่ละ request

from anthropic import Anthropic
client = Anthropic()
messages = []

def chat(user_text):
    messages.append({"role": "user", "content": user_text})
    resp = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=messages,
    )
    messages.append({"role": "assistant", "content": resp.content})
    return resp
3

นิยาม tool ด้วย JSON schema

บอก Claude ว่ามีเครื่องมืออะไร ใช้เมื่อไหร่ ผ่าน schema

tools = [{
  "name": "get_weather",
  "description": "ดึงสภาพอากาศปัจจุบันของเมืองที่ระบุ",
  "input_schema": {
    "type": "object",
    "properties": {"city": {"type": "string"}},
    "required": ["city"],
  },
}]
4

วน loop: เรียก tool → ส่งผลกลับ

ถ้า Claude ขอใช้ tool (stop reason = tool_use) ให้ execute ฟังก์ชันจริง แล้ว ส่งผลกลับเป็น message ให้มันตอบต่อ — วนจนได้คำตอบสุดท้าย

resp = client.messages.create(model="claude-sonnet-4-6",
    max_tokens=1024, messages=messages, tools=tools)

while resp.stop_reason == "tool_use":
    for block in resp.content:
        if block.type == "tool_use":
            result = get_weather(**block.input)   # ฟังก์ชันจริงของเรา
            messages.append({"role": "assistant", "content": resp.content})
            messages.append({"role": "user", "content": [{
                "type": "tool_result",
                "tool_use_id": block.id,
                "content": str(result),
            }]})
    resp = client.messages.create(model="claude-sonnet-4-6",
        max_tokens=1024, messages=messages, tools=tools)
✅
แชตบอตคุยต่อเนื่องได้ และเมื่อถามเรื่องอากาศ มันจะเรียก get_weather เองแล้วตอบด้วยข้อมูลจริง
5

ต่อยอด

  • streaming เพื่อแสดงผลทีละ chunk แก้ปัญหา response ช้า
  • prompt caching เมื่อ system prompt/บริบทยาวและส่งซ้ำ ช่วยเร็ว+ถูกลง
  • เพิ่ม built-in tools เช่น web search; หรือทำ RAG ถ้าต้องอ้างอิงเอกสารเยอะ
💡
ชื่อโมเดล/พารามิเตอร์ล่าสุดให้ยึดตามเอกสารทางการที่ docs.anthropic.com

🎯 สิ่งที่ได้จากเคสนี้

  • API เป็น stateless — เก็บ messages เองแล้วส่งทั้งชุดทุกครั้ง
  • tool use: นิยาม tool ด้วย JSON schema แล้ววน loop จน stop reason จบ
  • ส่งผลของ tool กลับเป็น tool_result ให้ Claude ตอบต่อ
  • ต่อยอดด้วย streaming, prompt caching, web search, RAG
