Case Study: Hook
auto-format โค้ด + บล็อกคำสั่งอันตราย

โจทย์: อยากให้โค้ดถูก format อัตโนมัติทุกครั้งที่ Claude แก้ไฟล์ และ กันไม่ให้รันคำสั่งอันตราย/อ่านไฟล์ลับ — ใช้ Hook ที่รันแน่นอนทุกครั้ง

🎯 ใช้กับ Claude Code🧩 อิงคอร์ส Claude Code in Action📋 step-by-step
🧭
สถานการณ์ Hook คือคำสั่งที่รันในจุดของ lifecycle จุดเด่นคือ deterministic — รันแน่นอนทุกครั้ง ไม่ต้องหวังให้โมเดล "จำ" ทำเอง เคสนี้ใช้ 2 hook: PostToolUse (format หลังแก้ไฟล์) และ PreToolUse (กันก่อนทำ)

1

PostToolUse — format อัตโนมัติหลังแก้ไฟล์

ทุกครั้งที่ Claude ใช้ tool แก้ไฟล์ (Edit/Write) ให้รัน formatter ทันที โค้ดจะสม่ำเสมอโดยไม่ต้องสั่ง ตั้งค่าในไฟล์ settings.json

# .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command",
            "command": "npx prettier --write \"$CLAUDE_FILE_PATHS\"" }
        ]
      }
    ]
  }
}
💡
อยากจับ type error ด้วย? เพิ่มคำสั่งให้รัน tsc --noEmit ใน hook เดียวกัน เพื่อเตือนตรงจุดที่แก้
2

PreToolUse — บล็อกก่อนทำสิ่งอันตราย

PreToolUse รัน ก่อน tool ทำงาน จึง บล็อกได้ เคสนี้กันการอ่านไฟล์ความลับอย่าง .env

# สคริปต์รับ JSON ทาง stdin แล้วใช้ exit code บอกผล
# .claude/hooks/block-env.sh (ใช้ absolute path เสมอ)
#!/usr/bin/env bash
input=$(cat)
if echo "$input" | grep -q '\.env'; then
  echo "บล็อก: ห้ามอ่านไฟล์ .env" >&2
  exit 2   # exit code != 0 = บล็อก + ส่ง feedback กลับให้ Claude
fi
exit 0
3

เสริมด้วย permissions.deny ให้ครอบคลุม

Hook อย่างเดียวอาจไม่ครอบทุกช่องทาง (Grep/Bash) — เสริมด้วยกฎ permissions เพื่อความปลอดภัยอีกชั้น

{
  "permissions": {
    "deny": ["Read(**/.env)", "Read(**/.env.*)"]
  }
}
4

แชร์ให้ทีมอย่างปลอดภัย

ข้อควรระวังจากคอร์ส: ใช้ absolute path ใน hook (กัน path interception) และแชร์ผ่าน settings.example.json + placeholder $PWD + setup script ให้คนในทีมรันตั้งค่าเอง

🔧
event อื่น ๆ ที่มี: นอกจาก Pre/PostToolUse ยังมี Notification, Stop, SubagentStop, PreCompact, UserPromptSubmit, SessionStart, SessionEnd · ตรวจ syntax ล่าสุดที่ code.claude.com/docs/en/hooks

🎯 สิ่งที่ได้จากเคสนี้

  • Hook ทำงาน deterministic — รันแน่นอนทุกครั้งในจุดของ lifecycle
  • PostToolUse (matcher Edit|Write) = auto-format / ตรวจ type หลังแก้
  • PreToolUse บล็อกได้ — กันอ่าน .env / คำสั่งอันตราย ผ่าน exit code
  • เสริมด้วย permissions.deny และใช้ absolute path เสมอ
